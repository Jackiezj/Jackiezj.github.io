Hexo

1. 什么是Hexo

快速、简洁的博客框架，Markdown渲染

2. 安装

安装前提：
Node.js
Git

3. 建站

1) _config.yml

网站的配置信息

2）package.json

应用程序的信息。EJS, Stylus 和 Markdown renderer 已默认安装，您可以自由移除。

3) scaffolds

模版文件夹
当您新建文章时，Hexo 会根据 scaffold 来建立文件。
Hexo的模板是指在新建的markdown文件中默认填充的内容。例如，如果您修改scaffold/post.md中的Front-matter内容，那么每次新建一篇文章时都会包含这个修改。

4）source

资源文件夹是存放用户资源的地方。除 _posts 文件夹之外，开头命名为 _ (下划线)的文件 / 文件夹和隐藏的文件将会被忽略。Markdown 和 HTML 文件会被解析并放到 public 文件夹，而其他文件会被拷贝过去。

  • 资源文件夹

5）themes

主题 文件夹。Hexo 会根据主题来生成静态页面。

4. Hexo指令

主题

1) MiHo主题

安装和更新

配置

  • 站点配置文件 _configl.yml
  • 主题配置 themes/miho/_config.yml

文章封面图
cover_picture: images/banner.jpg
背景粒子
open_bg_particle: true
开启主页及头部动画
open_animation: true
评论

支持畅言、友言、disqus。

数据统计

支持站长、百度、google三种数据统计，正确填写配置信息即可。

文章基本配置

创建 about 页面
在 Hexo 目录执行如下命令：

hexo new page about
然后编辑source/about/index.md文件即可，可以参考我写好的[about]页面。
