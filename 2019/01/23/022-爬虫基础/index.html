爬虫

###　爬虫基础

1. 什么是网络爬虫

  • 简介

    12306抢票, 网站头票, 短信轰炸

  • Robots协议:

    网络爬虫排除标准: 网站通过Robots协议告诉搜索引擎哪些页面不能抓取, 哪些页面可以抓取

2. 通用爬虫和聚焦爬虫

1) 通用爬虫

  • 简介

    • 搜索引擎的抓取系统, 互联网 –> 本地 –> 形成内容镜像 (baidu, google)

  • 抓取流程

    1. 抓取网页 : start –> 初始URL –> 获取网页 –> 进取新的URL放入URL队列 –> 循环 –> end
    2. 数据存储 : 将HTML内容存入数据库, 如果网页内容大量重复可能不用抓取
    3. 预处理 : 提取文字, 中文分词, 消除噪音等
    4. 提供检索服务,网站排名

2) 聚焦爬虫

  • 简介

    • 在网页抓取时对内容进行处理筛选, 尽量保证只抓取需求相关的网页信息 (优于通用爬虫)

  • 抓取流程

3. HTTP和HTTPS

  • 简介

    HTTP协议: 发布和接收HTML页面的方法 端口: 80

    HTTPS协议: HTTP+SSL 在传输层保障数据传输安全 端口: 443

1) HTTP通信

  • 客户端请求信息

    请求内容 : 请求行, 请求头, 空行, 请求数据

    请求行 : 请求方法 URL 协议版本 \r\n GET https://www.baidu.com/ HTTP/1.1

    请求头 : 头部字段名 : 值 –> 请求头 Host: www.baidu.com

    … Cookie: UID=04A7B:FG=1; SID=34EA74A

    \r\n\r\n \r\n

    请求体 content

  • 请求方法

    | 序号 | 方法 | 描述 |
    | —- | ——- | ———————————————————— |
    | 1 | GET | 请求指定的页面信息，并返回实体主体。 |
    | 2 | HEAD | 类似于get请求，只不过返回的响应中没有具体的内容，用于获取报头 |
    | 3 | POST | 向指定资源提交数据进行处理请求（例如提交表单或者上传文件），数据被包含在请求体中。POST请求可能会导致新的资源的建立和/或已有资源的修改。 |
    | 4 | PUT | 从客户端向服务器传送的数据取代指定的文档的内容。 |
    | 5 | DELETE | 请求服务器删除指定的页面。 |
    | 6 | CONNECT | HTTP/1.1协议中预留给能够将连接改为管道方式的代理服务器。 |
    | 7 | OPTIONS | 允许客户端查看服务器的性能。 |
    | 8 | TRACE | 回显服务器收到的请求，主要用于测试或诊断。 |

    • GET请求与POST请求

      GET从服务器获取数据, POST请求向服务器传送数据

      GET请求参数显示在URL中, POST请求参数在请求体中

  • 常用请求报头

    • Host请求报头 : 指定web名称和端口 Host: www.baidu.com # 默认80

    • Connection : 链接类型 Connection : keep-alive

      Server支持keep-alive回复Connection : keep-alive响应, 不关闭连接, 直到一方先关闭

      Server不支持keep-alibe回复Connection : close 响应, 关闭连接

    • Upgrade-Insecure-Requests : 升级为HTTPS, HTTPS承载的页面不能出现HTTP请求

    • User-Agent : 浏览器名称

    • Accept : 表示客户端接收个么样的数据

      • Accept: */* 表示什么都可以接收
      • Accept: image/gif 表示客户端接收Gif图像格式资源
      • Accept: text/html 表示主客户端支持html文本
      • Accept: application/xhtml+xml 表示支持xhtml和xml格式的资源
      • Accept: text/html, application/xhtml+xml;q=0.9, image/*;q=0.8
        • q表示权重 0 <= q <= 1, q越大,越倾向于获得;前面的值

    • Referer: https://translate.google.cn/ 表示页面跳转处

    • Accept-Encoding : 文件编码格式

    • Accept-Language : 表示页面语言为us,zh-cn

    • Accept-Charset : 字符编码 utf-8等

    • Cookie : 在浏览器中寄存的小型数据体, 记载服务器相关用户信息

    • Content-Type : POST请求体中内容类型 eg: Content-Type = Text/XML; charset=gbk2312
